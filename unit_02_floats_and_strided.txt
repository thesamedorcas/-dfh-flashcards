sign bit;the leftmost bit — 0 = positive, 1 = negative → float 0 11111111 00000000... is negative infinity
mantissa (or significand);the fractional bits representing precision (implicit leading 1 assumed) → e.g. mantissa 101... represents 1.101 in binary = 1.625
exponent;the bits that set the scale of the number → stored as unsigned int with a bias subtracted to get the true exponent
exponent bias;added to the stored exponent so negative exponents can be stored as positive bits → float32 bias = 127, so stored 130 means true exponent 3 (= 2^3 = 8)
strided array;data stored flat in memory, but a header (dope vector) with shape and strides tells NumPy how to interpret it as multi-dimensional → memory: [1,2,3,4,5,6], shape (2,3), strides (3,1) gives $$\begin{bmatrix}1&2&3\\4&5&6\end{bmatrix}$$
strides in NumPy;number of elements to skip to move one step in each dimension → e.g. a (2,3) float64 array: strides = (24, 8) bytes — 24 bytes to jump a row (3×8), 8 bytes to jump a column
C ordering (row-major);rows stored consecutively in memory — rightmost index changes fastest → $$\begin{bmatrix}1&2&3\\4&5&6\end{bmatrix}$$ stored as [1, 2, 3, 4, 5, 6]
Fortran ordering (column-major);columns stored consecutively in memory — leftmost index changes fastest → $$\begin{bmatrix}1&2&3\\4&5&6\end{bmatrix}$$ stored as [1, 4, 2, 5, 3, 6]
dope vector;the array header containing shape, strides, dtype, and data pointer — this is all NumPy changes for O(1) operations like transpose
advantages of C ordering;efficient for row-wise traversal (cache-friendly) — iterating along a row hits consecutive memory addresses
advantages of Fortran ordering;efficient for column-wise traversal — suits LAPACK and mathematical libraries that think column-first
dope vector for a 3D array;e.g. shape=(2,3,4), strides=(12,4,1): to get element [i,j,k], compute offset = i×12 + j×4 + k×1
effect of changing strides on array iteration;different strides make the same flat data look like a different shape — transposing just swaps stride values, no data moves
Fortran ordering in nested arrays;innermost arrays are stored column-by-column — required by Fortran-based libraries like LAPACK
Illiffe vector;alternative to dope vector: an array of pointers to rows (like a C array of arrays) — supports ragged shapes but is slower and less cache-friendly
IEEE 754 exceptions;special conditions during floating-point arithmetic: overflow, underflow, division by zero, invalid operation, inexact
overflow;result exceeds the representable range → if untrapped, becomes ±Inf → e.g. float32 max ≈ 3.4×10³⁸, anything larger overflows
underflow;result is too close to zero to represent as a normalised float → if untrapped, becomes ±0 → e.g. a very small product like 1e-300 × 1e-200 in float64
division by zero;nonzero ÷ 0 → if untrapped, produces ±Inf → e.g. 1.0 / 0.0 = Inf, -1.0 / 0.0 = -Inf
invalid operation;mathematically undefined → produces NaN → e.g. √(-1), 0/0, ∞ − ∞, 0 × ∞
trapping exceptions;IEEE 754 lets you register a handler so that instead of silently producing NaN/Inf, an interrupt fires — useful for debugging
setting traps in IEEE 754;use np.seterr(divide='raise') to make division by zero raise a Python exception instead of returning Inf
handling NaN (Not a Number);NaN represents undefined results — propagates: any operation with NaN returns NaN → NaN + 1 = NaN, NaN == NaN is False
handling Inf (Infinity);±Inf represents overflow or 1/0 — Inf + 1 = Inf, 1/Inf = 0, Inf - Inf = NaN
detecting and avoiding NaN;np.isnan(x) returns True where NaN → replace with: x[np.isnan(x)] = 0 or use np.nanmean() etc.
using Inf in computations;can serve as a sentinel "infinity" value — e.g. initialise min-so-far = +Inf, then any real value will be smaller
NaN (Not a Number);special bit pattern: all-1 exponent + non-zero mantissa → represents undefined results, poisons any arithmetic it touches
propagation of NaN;NaN spreads through computation — if any input is NaN, the output is NaN → NaN + 5 = NaN, max(3, NaN) = NaN
NaN in comparisons;NaN == NaN is False; NaN != NaN is True — use np.isnan() not == to check for NaN
isnan() function;np.isnan(x) returns a boolean array — True where NaN, False elsewhere → np.isnan([1.0, nan, 3.0]) = [F, T, F]
NaN as a placeholder;used to mark missing data in arrays — e.g. temperature sensor dropout stored as NaN instead of -999
handling NaN in data cleaning;np.nanmean(), np.nansum() etc. skip NaNs; or x[~np.isnan(x)] filters them out
IEEE 754 standard for NaN;all-1 exponent bits with any non-zero mantissa → many NaN bit patterns exist ("NaN payload"), but all behave the same way in arithmetic
NaN in mathematical functions;functions return NaN for invalid inputs → np.sqrt(-1.0) = NaN, np.log(-1.0) = NaN
NaN in statistical calculations;np.mean([1, nan, 3]) = NaN; use np.nanmean([1, nan, 3]) = 2.0 to ignore NaNs
NaN (Not a Number) bit pattern;64-bit float: [sign=any][exponent=all 1s = 11111111111][mantissa=non-zero]
Inf (Infinity) bit pattern;64-bit float: [sign=0 or 1][exponent=all 1s = 11111111111][mantissa=all 0s] → +Inf and −Inf differ only in the sign bit
Positive zero bit pattern;64-bit float: [0][all zeros] — sign=0, exponent=0, mantissa=0
Negative zero bit pattern;64-bit float: [1][all zeros] — sign=1, but +0 == -0 is True in IEEE 754
Signed zero issue;+0 and −0 compare equal but 1/(+0) = +Inf while 1/(−0) = −Inf — can cause surprising sign flips
Reshape for Promotion;np.reshape(x, new_shape) rearranges dimensions — e.g. shape (6,) → (2,3) → $$\begin{bmatrix}0&1&2\\3&4&5\end{bmatrix}$$
Squeeze for Reduction;np.squeeze(x) removes all size-1 axes → shape (3,1,4,1) → (3,4); useful after reductions that leave singleton dims
Newaxis for Promotion;np.newaxis inserts a size-1 axis → x[:,np.newaxis] turns shape (3,) into (3,1) so it can broadcast against a (1,4) array
None for Promotion;x[:,None] identical to x[:,np.newaxis] — inserts a size-1 axis at that position
Ravel for Flattening;np.ravel(x) returns a 1-D view of x in C order → $\begin{bmatrix}1&2\\3&4\end{bmatrix}$ → [1, 2, 3, 4] (view, not copy)
Flatten for Flattening;x.flatten() returns a 1-D copy → always allocates new memory unlike ravel which may return a view
Einsum Function;np.einsum(subscripts, A, B) — Einstein summation: concise notation for dot products, matrix mults, transposes, traces
Einsum Notation;letters label axes; repeated letters are summed over → 'ij,jk->ik' is matrix multiply; 'ii->' is trace
Einsum Example;np.einsum('ij->ji', A) transposes A — just swaps the axis labels; 'ik,kj->ij' is standard matrix multiply
Swapaxes Function;np.swapaxes(arr, axis1, axis2) swaps two axes — e.g. shape (2,3,4) swapaxes(0,2) → shape (4,3,2)
Swapaxes Example;np.swapaxes(arr, 0, 1) on shape (2,3) gives shape (3,2) — equivalent to transpose for 2D
np.meshgrid();creates coordinate grids from 1-D vectors → X,Y = np.meshgrid([1,2,3],[4,5]) gives two 2×3 arrays for vectorised evaluation of f(x,y)
Benefits of Einsum;avoids temporary arrays, can express complex multi-axis contractions in one line — often faster than chains of np.dot/np.sum
Machine Epsilon;smallest ε such that 1.0 + ε ≠ 1.0 — marks the limit of floating point precision relative to 1.0
Double Precision Machine Epsilon;float64: ε ≈ 2.22×10⁻¹⁶ — values closer than this to 1.0 cannot be distinguished from 1.0
Single Precision Machine Epsilon;float32: ε ≈ 1.19×10⁻⁷ — about 7 significant decimal digits
IEEE 754 Roundoff Guarantees;for +, −, ×, ÷, √: result is the nearest representable float to the exact answer — error ≤ 0.5 ULP
Roundoff Guarantee Example;1.0 + 2.0 = exactly 3.0; 0.1 + 0.2 ≠ 0.3 because 0.1 and 0.2 are not exactly representable — the error is < 1 ULP
IEEE 754 Rounding Modes;round to nearest (default), round toward +∞, round toward −∞, round toward zero — most code uses round-to-nearest
ULPs (Units in the Last Place);the gap between a float and its nearest neighbour — ULP(1.0) ≈ 2.2×10⁻¹⁶ for float64; ULP grows with magnitude
Significance of ULPs;two floats differing by 1 ULP are as close as possible — use ULP distance to judge if a computation is "accurate"
Floating-Point Numbers and ULPs;between 1.0 and 2.0 there are 2⁵² representable float64 values — each step is 1 ULP ≈ 2.2×10⁻¹⁶
np.nextafter Function;returns the next representable float after x toward y → np.nextafter(1.0, 2.0) = 1.0000000000000002 (1 ULP above 1.0)
np.nextafter Example;np.nextafter(0.0, 1.0) returns the smallest positive float64 ≈ 5×10⁻³²⁴
np.nextafter Usage;use to probe float spacing, test edge cases, or demonstrate that 1.0 + eps/2 == 1.0
Avoiding Catastrophic Cancellation;when x ≈ y, x − y loses all significant digits — e.g. 1.000000001 − 1.000000000 = 1×10⁻⁹ but may have only 1 accurate digit
Comparing Floating-Point Numbers;never use ==; instead use np.allclose(a, b, rtol=1e-5, atol=1e-8) which checks |a−b| ≤ atol + rtol×|b|

What three parts make up a floating point number?;sign bit (1 bit) + exponent (biased integer) + mantissa (fractional, with implicit leading 1) → e.g. float32 = [s][8 exp bits][23 mantissa bits]
What is the implicit leading 1 in floating point?;mantissa is always in [1.0, 2.0), so the leading 1 is never stored — it is reconstructed automatically → frees one extra bit of precision
What is the exponent bias in float32?;127 — stored value minus 127 gives the true exponent → stored 130 means 2^(130−127) = 2^3 = 8
How many bits are sign/exponent/mantissa in float32?;1 sign + 8 exponent + 23 mantissa = 32 bits total
How many bits are sign/exponent/mantissa in float64?;1 sign + 11 exponent + 52 mantissa = 64 bits total
What integers can float64 represent exactly?;all integers from −2⁵³ to +2⁵³ — beyond that, consecutive integers can no longer be distinguished
What is cancellation error?;when x ≈ y, x − y loses significant digits → e.g. 1.000001 − 1.000000 = 1×10⁻⁶ but you may only have 1 accurate digit left
What is magnitude error?;when |x| >> |y|, adding y to x loses y's precision → e.g. 10¹⁵ + 1.0 may just return 10¹⁵ because 1.0 is below the ULP of 10¹⁵
Why must you never use floating point for financial calculations?;rounding errors accumulate — 0.1 + 0.2 ≠ 0.3 in float; use decimal types with defined rounding rules instead
How does transposing work using strides?;swap the stride values and shape values in the dope vector header — the data stays put, O(1)
How does reversing an array work using strides?;move the data pointer to the last element and negate the stride — no data is copied
What is the difference between a dope vector and an Illife vector?;dope vector: flat data + stride metadata (rectangular, cache-friendly, O(1) transpose); Illife: nested pointers (supports ragged arrays but slower)
What does np.allclose(x, y) do?;checks |x−y| ≤ atol + rtol×|y| element-wise — the correct way to compare floats → np.allclose(0.1+0.2, 0.3) = True
What does np.isfinite do?;returns True where element is neither NaN nor ±Inf → np.isfinite([1.0, nan, inf]) = [T, F, F]
What is bfloat16?;16-bit float used in ML hardware: 1 sign + 8 exponent + 7 mantissa — same exponent range as float32 but 3× less precision
How is a rigid transformation (flip, transpose) O(1)?;it only modifies the strides and shape in the dope vector header — the underlying data buffer is never touched
