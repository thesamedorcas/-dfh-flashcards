representing edge-weighted graphs as matrices; use an adjacency matrix where A[i,j] = weight of edge from i to j → e.g. 3-node graph: $$A = \begin{bmatrix}0&1&1\\1&0&1\\1&1&0\end{bmatrix}$$
spectral methods; representing graphs as matrices and studying their eigenvalues/eigenvectors to reveal structure
discrete-continuous interchange; discrete objects (graphs, words) can be represented as continuous matrices, enabling calculus-based methods
flow in a graph; multiply adjacency matrix by a flow vector → Av gives updated flow at each node → e.g. $\begin{bmatrix}0&1\\1&0\end{bmatrix}\begin{bmatrix}3\\5\end{bmatrix} = \begin{bmatrix}5\\3\end{bmatrix}$
sink; a node where more flow exits than enters — net outflow
source; a node where more flow enters than exits — net inflow
degree; number of edges connected to a node → in the matrix, it's the row sum (for unweighted graphs)
in-degree; number of edges entering a node — column sum of adjacency matrix
out-degree; number of edges leaving a node — row sum of adjacency matrix
adjacency matrix; square matrix where A[i,j] = weight of edge i→j; usually sparse (mostly zeros) → $$\begin{bmatrix}0&2&0\\0&0&3\\1&0&0\end{bmatrix}$$ means edge 0→1 weight 2, edge 1→2 weight 3, edge 2→0 weight 1
directed graph; edges have direction (arrow) — adjacency matrix is generally asymmetric: A[i,j] ≠ A[j,i]
undirected graph; edges have no direction — adjacency matrix is symmetric: A[i,j] = A[j,i]
flow conservation; total flow into a node = total flow out — represented by doubly stochastic matrices
stochastic matrices; each row is a probability distribution (non-negative, sums to 1) → $$\begin{bmatrix}0.7&0.3\\0.4&0.6\end{bmatrix}$$ rows sum to 1
row-stochastic; each row sums to 1 — represents probabilities of transitioning OUT of each state
column-stochastic; each column sums to 1 — represents probabilities of transitioning INTO each state
doubly stochastic matrices; both rows and columns sum to 1 — conserves flow in AND out of every node
matrix exponentiation; $A^5 = A @ A @ A @ A @ A$ — applying a linear transform 5 times in a row
integer matrix powers as repeated linear transformations; $A^n$ applies the transformation encoded by A exactly n times → $A^2$ = "two steps on a graph"
matrix inversion; $A^{-1}$ undoes A's transformation: $A^{-1}Ax = x$ → e.g. if A doubles everything, $A^{-1}$ halves everything
Number of eigenvalues; an N×N matrix has exactly N eigenvalues (counting complex ones and multiplicities)
Number of eigenvectors; an N×N matrix has N eigenvectors — may not all be distinct if the matrix is singular
Eigenvalues of a matrix; scalars λ satisfying det(A − λI) = 0 — the values that scale (not rotate) their corresponding eigenvector
Eigenvectors of a matrix; non-zero vectors v satisfying Av = λv — they pass through a linear transform unchanged in direction (just scaled by λ)
Interpretation of an eigenvector; the "special directions" of a transformation that don't rotate — just stretch or flip → e.g. for a scaling matrix, the axes are eigenvectors
Interpretation of an eigenvalue; how much the eigenvector is stretched (λ>1), compressed (0<λ<1), or flipped (λ<0) by the transformation
Trace of a matrix; sum of diagonal entries = sum of all eigenvalues → $\text{tr}\begin{bmatrix}3&1\\0&5\end{bmatrix} = 3+5 = 8$
Determinant; product of all eigenvalues → $\det\begin{bmatrix}2&0\\0&3\end{bmatrix} = 2\times3 = 6$ — geometrically: how much the transform scales area (2-D) or volume (3-D)
Geometric interpretation of the determinant; area of parallelogram spanned by columns of A — det=2 means the transform doubles area; det=0 means it collapses to a lower dimension
Singular; det(A) = 0 — the transform collapses at least one dimension; not invertible → e.g. $\begin{bmatrix}1&2\\2&4\end{bmatrix}$ (row 2 = 2×row 1)
Non-singular; det(A) ≠ 0 — all dimensions preserved, matrix is invertible
Power iteration method; repeatedly apply A and normalise → $x_{n+1} = Ax_n / \|Ax_n\|$ — converges to the eigenvector with the largest |λ|
Eigenpair; a matched pair (λ, v) where Av = λv
Eigendecomposition; $A = Q\Lambda Q^{-1}$ where columns of Q are eigenvectors and Λ is diagonal with eigenvalues → only possible for diagonalisable matrices
Geometric interpretation of eigenvalues; λ>1: stretches along that direction; 0<λ<1: compresses; λ<0: flips direction; λ=0: collapses that dimension to 0
Geometric interpretation of singularity; det=0 → at least one eigenvalue is 0 → the transform squashes space flat in at least one direction → information is lost, can't be undone
Real eigenvalues; symmetric matrices always have real (not complex) eigenvalues — this is why covariance matrices are well-behaved
Complex eigenvalues; indicate rotation in the transformation — e.g. the 2-D rotation matrix has complex eigenvalues e^{±iθ}
Relation between trace and eigenvalues; $\text{tr}(A) = \sum_i \lambda_i$ — the diagonal sum equals the eigenvalue sum
Relation between determinant and eigenvalues; $\det(A) = \prod_i \lambda_i$ — product of all eigenvalues
Principal Component Analysis (PCA); finds the directions of greatest variance in a dataset by computing the eigenvectors of the covariance matrix — used to reduce dimensions while keeping the most information
Eigenvalue Decomposition in PCA; decompose the covariance matrix $C = Q\Lambda Q^T$ — eigenvectors Q give the principal directions, eigenvalues Λ give how much variance each direction captures
Eigenspectrum; eigenvalues ranked by absolute magnitude → large eigenvalue = important direction with lots of variance; small eigenvalue = noise
Principal Components; eigenvectors of the covariance matrix — form a new orthogonal coordinate system aligned with the data's natural axes of variation
Variance explained by Principal Components; eigenvalue of PC_k / sum of all eigenvalues = fraction of total variance captured by that component
Dimensionality reduction; project data onto a lower-dimensional subspace — e.g. keep top-3 PCs from 100-D data → preserve 90% of variance in 3 numbers
Dimensionality Reduction using PCA; project onto top-k eigenvectors → $\hat{X} = XQ_k$ where $Q_k$ contains only the k largest eigenvectors — best rank-k approximation
Limitations of PCA; assumes linear relationships; sensitive to outliers; ignores non-Gaussian structure; cannot capture curved manifolds
Choosing the Number of Principal Components; plot cumulative explained variance vs k — pick k where the curve bends ("elbow") or where cumulative variance exceeds 90-95%
Applications of PCA; image compression, face recognition, gene expression analysis, noise reduction, data visualisation
Left singular vectors; columns of U in A = UΣV^T — they are eigenvectors of AA^T and form an orthonormal basis for the output (row) space
Right singular vectors; columns of V in A = UΣV^T — eigenvectors of A^TA; form an orthonormal basis for the input (column) space
Singular Value Decomposition (SVD); any matrix $A = U\Sigma V^T$ where U and V are orthogonal (rotations) and Σ is diagonal (scalings) → decomposes any linear map into: rotate, scale, rotate
Singular Values in SVD; diagonal entries of Σ — always non-negative, sorted largest first; indicate how much each direction is stretched by the transformation
Interpreting U and V^T in SVD; V^T rotates the input; Σ scales each axis; U rotates into the output space → together: any linear map = rotate → scale → rotate
Pseudo-inverse using SVD; $A^+ = V\Sigma^+ U^T$ where $\Sigma^+$ replaces each non-zero diagonal with its reciprocal — gives the least-squares solution to Ax = b
Pseudo-Inverse; $A^+$ generalises matrix inverse to non-square and singular matrices → minimises $\|Ax - b\|^2$; used in least squares
Low-Rank Approximation using SVD; keep only the top-k singular values → $A_k = U_k\Sigma_k V_k^T$ — best rank-k approximation to A (minimises Frobenius norm of error)
Condition Number; ratio of largest to smallest singular value: $\kappa(A) = \sigma_\max / \sigma_\min$ — large κ means the matrix is ill-conditioned: small input changes cause large output changes
Applications in Recommendation Systems; factorize a user×item rating matrix via SVD → the top singular vectors capture latent factors (genres, styles); reconstruct missing ratings
Linear Systems; $A\mathbf{x} = \mathbf{b}$ — find x given A and b → geometrically: find the point that A maps to b
Consistent Linear System; has at least one solution — b lies in the column space of A
Independent Linear System; exactly one solution — A is square and invertible (det ≠ 0)
Dependent Linear System; infinitely many solutions — b lies in column space but A is rank-deficient; solution set is a line, plane, or higher-dim affine subspace
Solving Linear Systems using Matrix Inversion; $\mathbf{x} = A^{-1}\mathbf{b}$ — only for square invertible A; computationally expensive and numerically fragile for large systems
Matrix Rank; number of linearly independent rows (= columns) = number of dimensions the transformation preserves → rank of $\begin{bmatrix}1&2\\2&4\end{bmatrix}$ is 1 (row 2 = 2×row 1)
Full Rank; rank = number of rows = non-singular = invertible
Deficient Rank; rank < number of rows = singular = not invertible — some dimension is collapsed to 0
Null rank; rank = 0 — the zero matrix; maps everything to the origin
Calculation of Rank; count non-zero singular values from SVD — each non-zero σ corresponds to a preserved dimension
Rank and Eigenvalues; rank of a square matrix = number of non-zero eigenvalues
Rank and singular values; rank of any matrix = number of non-zero singular values
Adjacency Matrix; A[i,j] = 1 if edge between i and j → for 3 nodes with edges 0-1 and 1-2: $$A=\begin{bmatrix}0&1&0\\1&0&1\\0&1&0\end{bmatrix}$$
Directed Graphs; adjacency matrix is asymmetric — A[i,j]=1 means arrow FROM i TO j
Undirected Graphs; adjacency matrix is symmetric — if A[i,j]=1 then A[j,i]=1 (no direction)
Weighted Graphs; A[i,j] = edge weight (not just 0/1) → e.g. road network where weights are distances
Positive Definite Matrix; $\mathbf{x}^T A\mathbf{x} > 0$ for all non-zero x — all eigenvalues positive — always invertible — intuitively: like a bowl, curving upward in every direction
Positive Semidefinite Matrix; $\mathbf{x}^T A\mathbf{x} \geq 0$ — eigenvalues ≥ 0 — covariance matrices are always PSD
Negative Definite Matrix; $\mathbf{x}^T A\mathbf{x} < 0$ — all eigenvalues negative — like an upside-down bowl; always invertible
Negative Semidefinite Matrix; $\mathbf{x}^T A\mathbf{x} \leq 0$ — eigenvalues ≤ 0
Indefinite Matrix; has both positive and negative eigenvalues — saddle shape; some directions curve up, others curve down
Orthogonal matrix; $A^T A = I$ — columns are orthonormal — represents a pure rotation (preserves lengths and angles, det=±1)
Pure rotation matrix; orthogonal matrix with det=+1 — rotates without flipping
Pure scaling matrix; diagonal matrix — e.g. $\begin{bmatrix}2&0\\0&3\end{bmatrix}$ stretches x by 2, y by 3
Sparse matrix; mostly zero entries → e.g. a web graph with 1 billion pages: most pages link to only ~10 others, so 99.999% of entries are 0
Markov Chain; sequence of states where transition probabilities depend only on the current state — modelled as a column-stochastic transition matrix
Transition Matrix; square column-stochastic matrix where T[j,i] = P(next state = j | current state = i) — multiply by a state vector to get next-step probabilities
Low rank approximation; keep top-k singular values and vectors: $A_k = \sum_{i=1}^k \sigma_i \mathbf{u}_i \mathbf{v}_i^T$ — e.g. compressing a 1000×1000 image using only 10 singular values
Whitening; transform data to have zero mean and identity covariance: $\tilde{X} = \Sigma^{-1/2}(X - \mu)$ — removes correlations and equalises variances
Demeaning; subtract the mean from every sample: $\tilde{x}_i = x_i - \bar{x}$ — centres the dataset at the origin
Standardisation; demean then divide by standard deviation: $\tilde{x}_i = (x_i - \mu)/\sigma$ — each dimension then has mean 0 and variance 1
What is the Laplacian matrix of a graph?;$L = D - A$ where D is the diagonal degree matrix and A is the adjacency matrix → e.g. $$A=\begin{bmatrix}0&1&1\\1&0&0\\1&0&0\end{bmatrix}, D=\begin{bmatrix}2&0&0\\0&1&0\\0&0&1\end{bmatrix}, L=\begin{bmatrix}2&-1&-1\\-1&1&0\\-1&0&1\end{bmatrix}$$
What does the Fiedler value tell you?;second smallest eigenvalue of L — small Fiedler value → graph is barely connected (one cut disconnects it); large → highly connected
What is the Fiedler vector?;eigenvector for the second smallest eigenvalue of L — sign of each entry indicates which side of the optimal 2-way graph partition that node belongs to
What does the number of zero eigenvalues of the Laplacian tell you?;number of disconnected components — one zero eigenvalue = connected graph; k zero eigenvalues = k separate components
What is the condition number of a matrix?;$\kappa(A) = \sigma_\max / \sigma_\min$ — measures sensitivity of solution to small perturbations; large κ → ill-conditioned → round-off errors are amplified
What is deflation in eigenvalue computation?;after finding the leading eigenvector v1, subtract its contribution ($\sigma_1 u_1 v_1^T$) from the matrix — the next power iteration then finds the second eigenvector
What is the SVD decomposition A = U Sigma V^T?;U: left singular vectors (orthogonal), Σ: diagonal positive singular values, V^T: right singular vectors (orthogonal) → any matrix = rotate → scale → rotate
What does the SVD split any matrix transform into?;V^T rotates the input coordinate frame, Σ scales each axis independently, U rotates into the output coordinate frame
How do you compute A^(1/2) using SVD?;$A^{1/2} = U\Sigma^{1/2}V^T$ where $\Sigma^{1/2}$ replaces each diagonal σ with √σ — the matrix square root
How do you invert a matrix using SVD?;$A^{-1} = V\Sigma^{-1}U^T$ where $\Sigma^{-1}$ replaces each diagonal σ with 1/σ — fails if any σ = 0
What is the pseudo-inverse A+?;$A^+ = V\Sigma^+ U^T$ — generalises inversion to non-square/singular matrices; finds x minimising $\|Ax - b\|^2$
What does low rank approximation mean?;keep only top-k singular values/vectors → throws away small σ (noise) and keeps large σ (signal) → compressed representation of the matrix
What does the determinant being zero mean geometrically?;the transformation collapses at least one dimension — area (2-D) or volume (3-D) becomes 0; information is destroyed and the transform can't be undone
What is a stochastic matrix?;non-negative entries where each row (or column) sums to 1 — models probability transitions between states; multiplying by a probability vector gives the next distribution
What is the degree matrix D?;diagonal matrix where $D_{ii}$ = sum of edge weights attached to node i — used to form the Laplacian: L = D − A
What are the numerical problems with direct matrix inversion?;floating point errors accumulate with each multiply; ill-conditioned matrices amplify errors massively; inverse of a sparse matrix is dense (storage explosion)
Why is iterative solving preferred over direct inversion for linear systems?;iterative methods (e.g. conjugate gradient) find one specific solution Ax=b without forming A^{-1} — much cheaper and more numerically stable
